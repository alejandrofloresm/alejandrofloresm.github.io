Consola de Git

En esta ocasión aprenderás algunos de los comandos básicos de Git, así como algunos comandos de Linux.

Solo para los usuarios de Windows

1. Descarga el software Putty de aquí: http://the.earth.li/~sgtatham/putty/latest/x86/putty.exe

2. Al abrir putty verás la siguiente pantalla:

3. Coloca la IP 104.131.130.64 en el campo de Host Name

4. El sistema te pedirá que aceptes la conexión con el servidor, da click en Si.

5. Posteriormente coloca como nombre de usuario tu matrícula

6. El sistema te pedirá que ingreses tu contraseña, la contraseña que tienes asignada son los digitos después del 0 de tu matrícula, por ejemplo, si tu matrícula es a00984699, tu contraseña es 984699, si tu matrícula es a01984699 entonces tu contraseña es 1984699.

Ahora continúa desde el paso 3 de la siguiente sección. Para los siguientes pasos cuando veas $ significa que la instrucción se escribe en la consola (la ventana negra que tienes abierta).

Termina la parte de los usuarios de Windows

0. Para los siguientes pasos cuando veas $ significa que la instrucción se escribe en la consola.

1. El primer paso consiste en iniciar sesión en la consola, si tienes una computadora con Mac OS X puedes hacerlo abriendo la aplicación "Terminal" (la app viene instalada de fábrica) o si usas Linux solo abre la terminal, dentro de la consola escribe el siguiente comando. Tienes que sustituir [tu-matricula] por tu matrícula en minúsculas.

$ ssh [tu-matricula]@104.131.130.64

Por ejemplo, suponiendo que tu matrícula es B00984799 entonces el comando quedaría como:

$ ssh b00984799@104.131.130.64

2. El sistema te pedirá que ingreses tu contraseña, la contraseña que tienes asignada son los digitos después del 0 de tu matrícula, por ejemplo, si tu matrícula es a00984699, tu contraseña es 984699, si tu matrícula es a01984699 entonces tu contraseña es 1984699.

$ ssh [tu-matricula]@104.131.130.64
[tu-matricula]@104.131.130.64 password:

3. Una vez que estes a dentro del sistema, revisa el directorio en el que te encuentras, esto puedes hacerlo con el comando pwd:

$ pwd
/home/[tu-matricula]
Esto significa que estás en el directorio [tu-matricula] dentro de la carpeta /home, es como si en Windows estuvieras dentro de C:/Users/MiNombre/Documentos

4. Crea un directorio con el nombre de hola_mundo_tutorial para hacerlo utiliza el comando mkdir.

$ mkdir hola_mundo_tutorial

5. Verifica que el directorio se creó utilizando el comando ls con el parámetro -l.

$ ls -l
drwxrwxr-x 2 [tu-matricula] [tu-matricula] 4096 Mar 1 23:36 hola_mundo_tutorial

6. Navega al directorio hola_mundo_tutorial, para hacerlo utiliza el comando cd.

$ cd hola_mundo_tutorial

7. Crea un archivo de texto, para esto puedes ocupar la aplicacion nano.

$ nano

8. La aplicación nano se abre en la consola, en la primera línea escribe tú matrícula.

9. Una vez que hayas escrito tu matrícula, presiona Ctrl + o, Nota: se tiene que presionar la tecla Ctrl independientemente si estás en Mac OS X o Windows.

10. Nano te pedirá que pongas un nombre al archivo, colocale el nombre de hola_mundo.txt y presiona enter.

10. Finalmente presiona Ctrl + x para salir de Nano.

11. Si listas los archivos (ls -l) podrás ver que se ha creado el archivo hola_mundo.txt

$ ls -l
-rw-rw-r-- 1 [tu-matricula] [tu-matricula] 10 Mar 1 23:50 hola_mundo.txt

12. Ahora vamos a indicar que esta carpeta es un repositorio git, para hacerlo hay que ejecutar el comando git init esto creará una carpeta escondida con el nombre de .git, el tener la carpeta .git es suficiente para que git pueda identificar que ahí se encuentra un repositorio, de ahora en adelante todos los archivos y subcarpetas que se agreguen a esta carpeta (hola_mundo_tutorial) podrán agregarse al sistema de versiones.

$ git init
Initialized empty Git repository in /home/[tu-matricula]/hola_mundo_tutorial/.git/

13. Si listas los archivos ocultos de la carpeta (ls -la) podrás ver que se ha creado la carpeta .git.

$ ls -la
total 16
drwxrwxr-x 3 [tu-matricula] [tu-matricula] 4096 Mar 1 23:57 .
drwxr-xr-x 4 [tu-matricula] [tu-matricula] 4096 Mar 1 23:36 ..
drwxrwxr-x 7 [tu-matricula] [tu-matricula] 4096 Mar 1 23:57 .git
-rw-rw-r-- 1 [tu-matricula] [tu-matricula] 10 Mar 1 23:50 hola_mundo.txt

14. Ahora vamos agregar el documento que creamos al primer commit de nuestro repositorio. Un commit es un conjunto de cambios en el repositorio. Lo que nosotros vamos a hacer es indicarle a git que agregué este archivo al conjunto de cambios.
Para agregar el archivo utilizamos el comando git add [nombre de archivo]

$ git add hola_mundo.txt

15. Ahora que ya lo tenemos agregado al conjunto de cambios revisemos cual es el estatus de git, para esto podemos utilizar el comando git status.

$ git status
On branch master

Initial commit

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)

    new file: hola_mundo.txt

Observa que se menciona que se ha agregado un nuevo archivo.

16. Ahora le indicaremos al sistema que haga el commit, anteriormente solamente le habíamos señalado qué se iba agregar al commit, sin embargo este aún no se ha creado.

17. Para agregar el primer commit utiliza el comando git commit con el parámetro -m "texto de descripción del commit".

$ git commit -m "Se agregó el archivo hola_mundo.txt con los generales de información"
*** Please tell me who you are.

Run

  git config --global user.email "you@example.com"
  git config --global user.name "Your Name"

to set your account's default identity.
Omit --global to set the identity only in this repository.

fatal: empty ident name (for <a00984699@git-practice-delete-after-may.(none)>) not allowed

18. El sistema te indicará que falta definir el correo electrónico y tu nombre, para hacerlo ejecuta los comandos git config --global user.email "Tu correo@correo.com" y git config --global user.name "Tu nombre". No olvides que tienes que colocar tu correo electrónico y tu nombre, puedes usar el correo que quieras, por ejemplo:

$ git config --global user.email "alejandrofloresm@itesm.mx"
$ git config --global user.name "Alejandro Flores"

19. Ahora vuelve a intentar el commit y ya no tendrás problemas.

$ git commit -m "Se agregó el archivo hola_mundo.txt con los generales de información"
[master (root-commit) bde27ac] Se agregó el archivo hola_mundo.txt con los generales de información
 1 file changed, 1 insertion(+)
 create mode 100644 hola_mundo.txt

20. Si ejecutas el comando git status verás que no tienes ningún cambio pendiente.

$ git status
On branch master
nothing to commit, working directory clean

41. Abre el archivo hola_mundo.txt, podrás ver que tiene un contenido similar a este:

a00984699
<<<<<<< HEAD
Alejandro Flores
Mad Max: Fury Road
=======
Licenciatura en Administración de Tecnologías de Información
>>>>>>> d97760afddb7e6ec13c49d34c3ea9790cbb95ff9

Lo que está entre <<<<<<< HEAD y ======= es lo que tienes almacenado en tu repositorio local.

<<<<<<< HEAD
Alejandro Flores
Mad Max: Fury Road
=======

Mientras lo que está entre ======= y >>>>>>> d97760afddb7e6ec13c49d34c3ea9790cbb95ff9 es lo que está almacenado en el repositorio origen (en este caso, nuestro repositorio origen es lo que está guardado en github.com).

=======
Licenciatura en Administración de Tecnologías de Información
>>>>>>> d97760afddb7e6ec13c49d34c3ea9790cbb95ff9

Git no conoce cuál es la línea correcta, ya sea si la que está en tu repositorio local o bien la de tu repositorio origen, por lo tanto, el trabajo queda en manos de desarrollador.

En este caso queremos que el documento tenga lo siguiente: 1. Matrícula, 2. Nombre, 3. Carrera y 4. Película favorita.

42. Modifica el documento para que coincida con el requerimiento anterior, recuerda que tienes que eliminar las líneas: <<<<<<< HEAD, ======= y >>>>>>> d97760afddb7e6ec13c49d34c3ea9790cbb95ff9.
